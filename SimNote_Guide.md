**SimNote** 프로젝트는 클라우드 의존성을 없애고 사용자 중심의 로컬 네트워크 동기화를 지향하는 아주 흥미로운 컨셉이네요\! 보안과 프라이버시를 중요하게 생각하는 사용자들에게 큰 소구력이 있을 것 같습니다.  
이 프로젝트를 성공적으로 빌드하기 위한 단계별 가이드를 정리해 드립니다.

## ---

**1\. 기술 스택 추천: Flutter (플러터)**

안드로이드, iOS, Windows, macOS를 하나의 코드베이스로 대응하기 위해 **Flutter**를 강력히 추천합니다.

* **이유:** 구글이 만든 프레임워크로, 데스크톱(Win/Mac) 지원이 안정적이며 네이티브 수준의 성능을 제공합니다. 특히 로컬 DB 및 네트워크 통신 패키지가 풍부합니다.

## ---

**2\. 데이터 구조 및 저장소 설계**

클라우드 없이 로컬에서 데이터를 관리하므로, 각 기기 내부에 강력한 데이터베이스가 필요합니다.

### **데이터베이스 (Local DB)**

* **추천:** **Isar** 또는 **Hive**  
* **특징:** NoSQL 기반으로 객체를 직접 저장할 수 있어 속도가 매우 빠르고, 해시태그나 즐겨찾기 같은 인덱싱 작업에 유리합니다.  
* **구조 설계:**  
  * Notebook (Folder): 노트를 그룹화하는 최상위 객체.  
  * Note: 제목, 본문, 생성일, 수정일, 즐겨찾기 여부(bool).  
  * Tag: 해시태그 관리를 위한 별도 테이블 (Note와 N:M 관계).

## ---

**3\. 핵심 기능: 로컬 네트워크 동기화 (P2P)**

클라우드 없이 같은 Wi-Fi 환경에서 데이터를 주고받는 것이 이 프로젝트의 기술적 핵심입니다.

### **기기 탐색 및 연결**

* **mDNS (Multicast DNS):** nsd\_rosenbaum 같은 패키지를 사용하여 같은 네트워크 내의 SimNote 실행 기기를 자동으로 찾습니다. (마치 에어드랍이나 프린터 찾기와 비슷합니다.)  
* **인증:** 최초 연결 시 기기 간 PIN 번호를 입력하여 '인증된 기기' 리스트에 등록(기기 고유 ID 저장)합니다.

### **데이터 전송 방식**

1. **동기화 버튼 클릭:** 사용자가 버튼을 누르면 서버 역할을 할 기기와 클라이언트 역할을 할 기기가 정해집니다.  
2. **데이터 비교 (Diffing):** 각 노트의 lastModified 타임스탬프를 비교하여 더 최신 데이터를 판별합니다.  
3. **소켓 통신 (TCP/UDP):** Dart:io 라이브러리의 RawSecureSocket을 사용하여 데이터를 암호화하여 직접 전송합니다.

## ---

**4\. UI/UX 구성 요소**

사용자가 폴더와 태그를 직관적으로 관리할 수 있는 구조가 필요합니다.

* **사이드바 (Navigation Rail):** 노트북(폴더) 목록, 즐겨찾기, 전체 태그 클라우드 배치.  
* **검색 및 필터:** \#태그 검색 시 즉시 관련 노트를 보여주는 실시간 인덱싱 검색창.  
* **동기화 상태창:** 현재 같은 네트워크에 접속된 기기 목록(내 PC, 내 폰 등)과 마지막 동기화 시간 표시.

## ---

**5\. 개발 로드맵 (단계별 실행)**

| 단계 | 목표 | 주요 작업 |
| :---- | :---- | :---- |
| **1단계** | 로컬 메모장 완성 | Isar DB 연동, 폴더/태그 기반 노트 작성 기능 구현 |
| **2단계** | 기기 검색 구현 | mDNS를 이용해 같은 Wi-Fi 내 다른 기기 IP 주소 감지 |
| **3단계** | 데이터 전송 실험 | 두 기기 간 JSON 형태의 노트 데이터 송수신 테스트 |
| **4단계** | 동기화 로직 최적화 | 수정 시간 비교를 통한 데이터 덮어쓰기 방지 및 병합 로직 |
| **5단계** | 보안 강화 | 전송 데이터 AES-256 암호화 및 기기 인증 절차 추가 |

### ---

**💡 전문가의 팁**

로컬 동기화 방식에서 가장 골치 아픈 문제는 \*\*'충돌(Conflict)'\*\*입니다. 예를 들어, 폰과 PC에서 동시에 같은 노트를 수정하고 동기화하면 어떤 것을 우선할지 결정해야 합니다. 이를 위해 **CRDT(Conflict-free Replicated Data Type)** 개념을 살짝 공부해보시는 것을 추천합니다.  
우선 **폴더 구조를 가진 간단한 로컬 노트 기능**부터 시작해 볼까요? 프로젝트 생성을 위한 Flutter 명령어부터 도와드릴 수 있습니다. 구체적으로 어떤 기능부터 코딩해보고 싶으신가요?
